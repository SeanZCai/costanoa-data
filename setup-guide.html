Five minutes to set up, then it runs itself. You never paste a password, an API key, or any other kind of secret. The one thing you'll be asked for is a six-digit code we email you.

What this is

We now have a shared database of every meeting Costanoa takes: summaries, attendees, the companies. Once you're set up, anything you record in Granola lands in there every morning, tagged with your name. Which means we can finally ask Claude things like "which cybersecurity founders has Tony met in the last quarter?" and get a real answer, instead of all of us scrolling through Granola separately.

Your part is the three steps below. After that you can forget this exists.

You'll need

  • Claude Code on your laptop. The desktop app, not the website.
  • Granola already connected inside Claude. Go to Settings → Integrations → Granola in Claude Code and hit Connect.
  • Access to your @costanoa.vc inbox. We email a one-time code there.

Nothing else. No password, no API key, no token. If anything asks for one of those, it's a bug. DM Sean.

Setup: three things to type into Claude Code

1 Install the plugin ~30 seconds

Open Claude Code. In the chat box, paste this and hit enter:

/plugin marketplace add SeanZCai/costanoa-data

Then this:

/plugin install costanoa-data@costanoa-data

You should see a confirmation. If Claude asks you to authorize GitHub, do it. The repo is private, and that's how it checks that you're on the team.

After install, the new commands show up as /costanoa-data:granola-sync and /costanoa-data:team-onboard. Most of the time the short form /granola-sync works too. If the commands don't seem to register right away, run /reload-plugins once and they'll appear.

2 Confirm Granola is connected ~10 seconds

In the chat box, type:

who am I signed in as on Granola?

Claude should reply with your @costanoa.vc email. If it says nothing back, or shows a different email, head to Settings → Integrations → Granola in Claude Code and reconnect using your Costanoa Google account.

3 Run your first sync (the email-code part) ~2 to 5 minutes

Type:

/granola-sync

Six things happen, in order:

  1. Claude installs some helper files in the background. You'll see messages like "creating venv" or "installing dependencies." This takes about half a minute, and only happens on the first run.
  2. Claude asks: "What's your Costanoa email?" Type in your @costanoa.vc address.
  3. Claude tells you to check your inbox. Supabase has just emailed you a code.
  4. Open the email. Subject line is usually "Your code is XXXXXX". If it isn't there in half a minute, check spam.
  5. Copy the six digits, paste them into Claude.
  6. Claude says "authenticated, ready" and starts pulling your last 30 days of meetings. That takes another minute or three depending on how many you have.

When it finishes you'll see something like:

Synced 47 new meetings into Supabase (latest: May 10, 2026).
You only do the email-code thing once. The login stays on your laptop, and the daily auto-sync (next section) uses it without bothering you.

Make it run every morning

Once your first manual sync worked, type this in Claude Code:

/schedule create daily at 7am: /granola-sync

That's the whole automation. Starting the next morning, your meetings flow in at 7am. It runs in the cloud, so your laptop doesn't need to be on or awake.

To check on it later, ask Claude "show my scheduled jobs". To stop it, say "delete my granola-sync schedule".

How to know it worked

Fastest check: ask Claude "how many of my meetings are in the Costanoa knowledge base?". The number should roughly match your Granola count.

You can also run /granola-sync a second time right after the first. It should report "0 new meetings", which is the system correctly noticing nothing has changed since the last run.

What ends up in the database

  • The meeting title, the date, and the AI summary Granola wrote for it.
  • The attendees, plus whatever companies we can figure out they belong to.
  • For 1-on-1s where Granola only captured you on the invite, we take a best guess at the other person from the meeting title. So "Jigsaw <> Sean" still becomes a link to Jigsaw, just at lower confidence.

Calendar holds and "Busy" placeholders get skipped automatically. Only real meetings end up in there.

Keeping certain meetings private

Some meetings have no business being in the shared DB. Board sessions, performance reviews, confidential LP conversations, anything you want to keep just for yourself. You can tell the sync to skip those in two ways.

Skip them just this one time

Tell the skill what to drop when you run it:

/granola-sync excluding "board meeting", "perf review"

Any meeting whose title, summary, or transcript contains one of those phrases (case doesn't matter) gets dropped. The meeting stays in Granola, it just doesn't end up in Supabase.

Skip them every time, forever

Tell Claude:

add "board meeting" and "perf review" to my granola exclusions permanently

That writes the phrases into a small config file on your laptop. The daily 7am sync picks them up automatically. You can list them, add more, or clear them later by asking Claude.

Be specific with phrases. "Acme" would skip anything that mentions Acme, even in passing. "Acme board meeting" or "Acme LP review" is safer.

If something goes wrong

"Claude doesn't seem to know who I am in Granola"

The Granola integration isn't connected. In Claude Code go to Settings → Integrations → Granola → Connect and sign in with your Costanoa Google account.

"I never got the 6-digit code"

Check spam first. If it's still not there after a minute, ask Claude to resend it. If that also fails, message Sean. It's most likely an email-config issue on the Supabase side, not anything you did.

"It says I'm not a Costanoa email"

Either you're signed into Granola with a non-Costanoa Google account, or you typed a personal email at the prompt. Run /granola-sync again and use your @costanoa.vc address.

"Python not found" or "Python 3.10+ required"

You don't have a recent enough Python installed. On a Mac, open Terminal and run brew install python@3.12. Then run /granola-sync again. If that's gibberish, ask Sean to walk you through it.

"Plugin marketplace add failed"

You probably don't have read access to the private repo yet. Send Sean your GitHub username, he'll add you, and you can retry.

Anything else weird

Message Sean. Don't try to fix it yourself. If something is actually broken, it's more useful for him to see the original symptom than a patched-over version of it.

Questions go to Sean. So do bugs, weird data, and anything you wish this did but doesn't.
